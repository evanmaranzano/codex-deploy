# AI Dev Installer 安装说明

## 文档用途

这份说明用于帮助 Windows 用户完成 `AI Dev Installer` 的安装，并通过应用内流程准备 `Codex` 所需的基础环境。

如果你已经拿到安装包：

`AI Dev Installer_0.1.0_x64-setup.exe`

可以按照下面的步骤，从启动安装包到确认环境状态，完整走完一次安装流程。

## 这款安装器会做什么

`AI Dev Installer` 会检查并准备以下组件：

- Git
- Python
- Node.js
- CC Switch
- Codex

其中，Git、Python、Node.js、CC Switch 会通过安装包内置资源执行静默安装；如果系统已经检测到这些组件，应用会自动跳过对应阶段，不需要手动卸载重装。

## 开始前请先确认

开始前建议先确认下面几点：

1. 当前电脑为 Windows 系统。
2. 电脑可以联网。
3. 建议使用管理员权限运行安装器。
4. 安装过程中不要手动关闭安装窗口或应用窗口。

特别说明：

- 如果没有管理员权限，应用会在安装前检查阶段直接提示失败。
- `Codex` 的安装依赖 Microsoft Store / App Installer / `winget`，因此联网和系统组件状态都很重要。

## 第一步：启动安装包

找到安装包：

`AI Dev Installer_0.1.0_x64-setup.exe`

双击后会看到标准安装向导。建议直接按向导提示继续。

## 第二步：完成安装向导

安装向导主要包含以下几个动作：

1. 阅读欢迎页后点击 `Next`。
2. 确认安装位置。
3. 点击安装并等待第三方资源解压和程序写入完成。
4. 在完成页点击 `Finish` 启动 `AI Dev Installer`。

默认安装位置通常为：

`C:\Users\Administrator\AppData\Local\AI Dev Installer`

如果你没有特殊需求，建议保留默认位置。

## 第三步：认识应用主界面

打开 `AI Dev Installer` 后，页面主要分为几个区域：

- 顶部横幅区
- `安装进度`
- `组件状态`
- 操作按钮区
- `状态日志`

第一次打开时，最值得关注的是：

- `重新检测环境`
- `安装 Codex`
- `全部安装`
- `重试当前阶段`
- `重新执行全部安装`
- `状态日志`

## 第四步：先执行一次环境检测

建议先点击一次：

`重新检测环境`

这个动作会重新检查电脑里是否已经存在 Git、Python、Node.js、CC Switch、Codex。

如果某个组件已经存在，界面通常会显示：

- `已安装`
- 或在后续安装流程里显示为已跳过

这意味着应用会尽量避免重复安装。

## 第五步：推荐优先使用“全部安装”

如果你希望由安装器统一处理整个流程，推荐直接点击：

`全部安装`

当前应用会按以下顺序执行：

1. 执行安装前检查
2. 安装 Git
3. 安装 Python
4. 安装 Node.js
5. 安装 CC Switch
6. 刷新环境探测结果
7. 安装 Codex
8. 验证安装结果

说明：

- 已安装组件会自动跳过。
- 安装命令执行完成后，应用还会做一次最终验证，所以看到进度暂停几秒是正常现象。

## 第六步：怎样理解“安装 Codex”

界面中虽然有 `安装 Codex` 按钮，但当前版本并不是只执行单一的 Codex 安装动作。

它仍然会经过安装前检查、基础组件检查与环境刷新，然后再进入 `Codex` 安装和最终验证。

因此，更准确的理解方式是：

- `全部安装`：完整推荐流程
- `安装 Codex`：以 Codex 为目标的安装流程，但仍会检查并补齐前置依赖

## 第七步：安装过程中怎么看状态

安装期间可以重点观察这三个位置：

1. `安装进度`
2. `组件状态`
3. `状态日志`

页面中的阶段文案通常包括：

- `等待执行`
- `执行安装前检查`
- `安装 Git`
- `安装 Python`
- `安装 Node.js`
- `安装 CC Switch`
- `刷新环境变量`
- `安装 Codex`
- `验证安装结果`
- `安装完成`
- `安装失败`

安装过程中建议：

1. 不要连续点击多个安装按钮。
2. 如果日志仍在刷新，优先继续等待。
3. 不要在系统弹出安装窗口时手动将其关闭。

## 第八步：如何确认安装完成

一般可以通过下面几个信号判断：

1. `安装进度` 显示 100% 或接近完成。
2. 当前阶段显示为 `安装完成`。
3. `Codex` 显示为 `已安装`。
4. 其他基础组件显示为 `已安装` 或 `已跳过`。

如果你想再确认一次，可以再点：

`重新检测环境`

只要重新检测后 `Codex` 仍然显示为已安装，就可以认为安装结果已经生效。

## 第九步：失败后应该怎么处理

如果页面显示失败，建议按下面顺序处理：

1. 先看 `安装进度` 区域中的错误提示。
2. 再看 `状态日志` 最后几条内容。
3. 点击一次 `重新检测环境`。
4. 点击 `重试当前阶段`。
5. 如果问题仍未解决，再点击 `重新执行全部安装`。

需要特别说明的是：

- `重试当前阶段` 当前并不是严格意义上的断点续跑。
- 它会重新启动安装流程，再根据系统检测结果跳过已经满足条件的组件。

因此，如果你看到某些组件被快速跳过，这是正常现象。

## 常见情况说明

### 1. 提示未以管理员身份运行

处理方法：

1. 关闭当前程序。
2. 找到安装后的 `AI Dev Installer` 或安装包。
3. 右键选择“以管理员身份运行”。

### 2. 某个组件明明装过，页面仍提示未安装

建议先执行：

1. `重新检测环境`
2. 等待几秒
3. 再查看组件状态是否更新

### 3. Codex 安装阶段较慢

这通常是因为 `Codex` 安装依赖 Microsoft Store / App Installer / `winget`。

如果前面的离线组件都很快，而 `Codex` 阶段明显更慢，属于正常现象。只要日志仍在刷新，建议继续等待。

### 4. 提示无法启动 winget 或 Microsoft Store 相关安装失败

这类问题通常说明系统里的 Microsoft App Installer / Microsoft Store 组件状态异常。

建议处理方式：

1. 确认 Microsoft Store 可以正常打开。
2. 确认系统中 `winget` 可用。
3. 关闭应用后重新打开，再重新尝试安装。

### 5. 安装资源校验失败

如果安装器提示资源校验失败，通常不是操作步骤问题，而更像是安装包文件损坏。

建议重新获取安装包后再试一次。

## 最短操作版

如果你只想看最短步骤，可以按下面做：

1. 双击 `AI Dev Installer_0.1.0_x64-setup.exe`
2. 按安装向导完成安装
3. 打开 `AI Dev Installer`
4. 点击 `重新检测环境`
5. 点击 `全部安装`
6. 等待执行完成
7. 确认 `Codex` 显示为已安装
